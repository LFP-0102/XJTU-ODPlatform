> **作者**: ODPlatform Team

## 一、模块概述

模型评估子系统提供**单模型评估**和**多模型对比评估**两大核心能力，产出结构化的评估报告（JSON + CSV + Markdown），支持评估历史追踪和深度错误分析。

### 核心功能

| 功能 | 说明 |
|------|------|
| 单模型评估 | 在指定数据集上评估一个模型，生成完整评估报告 |
| 多模型对比 | 在同一数据集上评估多个模型，生成对比差异报告 |
| 混淆矩阵分析 | 提取每类 TP/FP/FN，诊断误检和漏检问题 |
| 类别性能排序 | 按 mAP50-95 / F1 / Precision / Recall 排名 |
| 问题类别诊断 | 自动标记严重/关注/正常类别 |
| 评估历史追踪 | 保存每次评估记录，生成指标趋势报告 |
| 类别级对比 | 对比两个模型在每个类别上的表现差异 |

### 模块文件结构

```
model_eval/
├── __init__.py      # 公共 API 导出
├── metrics.py       # EvalMetrics 数据结构（核心指标快照）
├── report.py        # EvalReport + ComparisonReport（报告生成）
├── service.py       # 编排服务（评估 + 对比 + 历史 + 分析）
├── analyzer.py      # ★ 新增：混淆矩阵分析 + 类别排名 + 问题诊断
└── history.py       # ★ 新增：评估历史追踪 + 趋势报告
```

## 三、设计决策与纪律

### 3.1 数据与展示分离

- **纯数据类**（frozen dataclass）：`EvalMetrics`、`EvalReport`、`ComparisonReport`、`ConfusionAnalysis`、`ClassRanking`、`EvalRecord`
  - 可以被单测直接断言
  - 可以 JSON 序列化
  - 不持有任何展示逻辑
- **渲染函数**：`render_markdown()`、`render_to_logger()`
  - 只消费数据，不持有数据
  - Markdown 输出可直接写入 `.md` 文件

### 3.2 复用优先（不造轮子）

- `EvalMetrics.from_yolo_results()` 复用 `TrainMetrics.from_yolo_results()` 抽取 ultralytics 原生指标
- `_safe_float()` 兜底 NaN，处理 ultralytics 跨版本字段变化
- `RunContext("evaluation")` 走统一的运行时目录管理

### 3.3 双轨输出

| 轨道 | 产物 | 用途 |
|------|------|------|
| 机器轨道 | `report.json` / `comparison.json` | 程序读取、CI 断言、下游工具 |
| 人工轨道 | `result.csv` / `comparison.csv` | Excel 打开、拖拽排序 |
| 可读报告 | `report.md` / `comparison.md` | 直接阅读、团队分享 |

### 3.4 评估历史设计

- 历史文件保存在 `runs/evaluation/_history/<dataset_name>_eval_history.json`
- 每个数据集一个历史文件，记录该数据集上所有模型的评估结果
- 重复 `run_id` 自动跳过（幂等）
- 历史保存失败不阻塞主流程（warn 而非 error）

## 四、使用指南

### 4.1 CLI 命令行

#### 基本语法

```bash
odp-eval (--model MODEL | --models MODEL [MODEL ...]) [选项]
```

#### 单模型评估

```bash
# 使用默认 val.yaml 配置评估 best.pt
odp-eval --model best.pt --data helmet_detection_v1

# 指定数据集划分和置信度阈值
odp-eval --model best.pt --data helmet_detection_v1 --split test --conf 0.5

# 使用 GPU 0，指定输入尺寸
odp-eval --model best.pt --data helmet_detection_v1 --device 0 --imgsz 1280

# 指定自定义 val 配置文件
odp-eval --model best.pt --data helmet_detection_v1 --config my_val_config.yaml
```

#### 多模型对比评估

```bash
# 对比两个模型
odp-eval --models yolo11n.pt yolo11s.pt --data helmet_detection_v1

# 对比多个模型（包括自训练权重）
odp-eval --models yolo11n.pt yolo11s.pt best.pt --data helmet_detection_v1 --split val

# 关闭图表生成以加速批量对比
odp-eval --models a.pt b.pt c.pt --data helmet_detection_v1 --no-plots

# 使用绝对路径
odp-eval --model /path/to/custom.pt --data /path/to/data.yaml
```

#### 所有 CLI 参数

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `--model` | str | — | 单个模型引用（与 `--models` 互斥） |
| `--models` | str[] | — | 多个模型引用（与 `--model` 互斥） |
| `--data` | str | None | 数据集名或 yaml 路径 |
| `--config` | str | `val` | 运行时 val 配置文件名 |
| `--no-archive` | flag | False | 不归档运行时配置 |
| `--split` | str | val.yaml | 数据集划分（train/val/test） |
| `--conf` | float | val.yaml | 置信度阈值 |
| `--iou` | float | val.yaml | NMS IoU 阈值 |
| `--imgsz` | int | val.yaml | 推理输入尺寸 |
| `--device` | str | val.yaml | 运行设备（cpu/0/0,1） |
| `--batch` | int | val.yaml | 批大小 |
| `--no-plots` | flag | — | 不生成评估图表 |
| `--log-level` | str | INFO | 日志级别（DEBUG/INFO/WARNING/ERROR） |

### 4.2 Python API

#### 单模型评估

```python
from pathlib import Path
from od_platform.model_eval import evaluate_model
from od_platform.runtime_config import build_val_config
from od_platform.common.run_context import RunContext
from od_platform.common import refs

# 1. 构建 val 配置
config, merger = build_val_config(yaml_path="val.yaml")

# 2. 解析数据集 yaml
data_yaml = refs.resolve_dataset_yaml("helmet_detection_v1")

# 3. 在 RunContext 中执行评估
with RunContext("evaluation") as run:
    result = evaluate_model(
        config=config,
        data_yaml=data_yaml,
        run=run,
        model_ref="best.pt",
        merger=merger,
        write_report=True,        # 落盘报告
        save_history=True,         # 追加到历史
        enhanced_analysis=True,    # 生成增强分析
    )

if result.success:
    print(f"评估完成: {result.report.model_name}")
    print(f"mAP50-95: {result.report.metrics.mAP50_95:.4f}")
else:
    print(f"评估失败: {result.message}")
```

#### 多模型对比评估

```python
from od_platform.model_eval import compare_models

with RunContext("evaluation") as run:
    result = compare_models(
        config=config,
        data_yaml=data_yaml,
        run=run,
        model_refs=["yolo11n.pt", "yolo11s.pt", "best.pt"],
        merger=merger,
        write_report=True,
        save_history=True,
        enhanced_analysis=True,
    )

if result.success:
    report = result.report
    print(f"总体最优模型: {report.overall_winner()}")
    print(f"最快模型: {report.fastest_model()}")
    # 查看各指标最佳
    for key in ["mAP50_95", "mAP50", "f1", "precision", "recall"]:
        print(f"  {key} 最佳: {report.best_model(key)}")
```

#### 深度分析

```python
from od_platform.model_eval import (
    rank_classes, diagnose_problem_classes, compare_per_class,
)

# 类别性能排序
rankings = rank_classes(metrics, sort_by="mAP50_95", top_n=5)
for r in rankings:
    print(f"#{r.rank} {r.class_name}: mAP50-95={r.mAP50_95:.4f}")

# 问题类别诊断
diag = diagnose_problem_classes(metrics, threshold=0.5)
print(f"严重: {diag['critical']}")
print(f"关注: {diag['warning']}")
print(f"正常: {diag['good']}")

# 两类模型在每个类别上的差异
diffs = compare_per_class(metrics_a, metrics_b, metric="mAP50_95")
for d in diffs:
    winner = "A" if d.winner == "A" else ("B" if d.winner == "B" else "持平")
    print(f"  {d.class_name}: diff={d.diff:+.4f} → {winner}")
```

#### 评估历史追踪

```python
from od_platform.model_eval.history import EvalHistory

# 加载现有历史
history = EvalHistory.load_or_create("datasets/helmet_detection_v1.yaml")

# 查看某模型的所有评估记录
records = history.get_records("my_model")
for r in records:
    print(f"{r.run_id}: mAP50-95={r.mAP50_95:.4f}")

# 查看最佳记录
best = history.best_record("my_model", metric="mAP50_95")
print(f"最佳 mAP50-95: {best.mAP50_95:.4f} (于 {best.run_id})")

# 趋势报告
trend = history.trend("my_model")
print(trend.render_markdown())
print(f"mAP50-95 改进: {trend.improvement('mAP50_95'):+.4f}")
```

## 五、增强分析说明

### 5.1 类别性能排序

在 `report.md` 中自动生成按 mAP50-95 降序排列的类别表，帮助快速定位：
- 表现最好的类别（模型对该类别识别最准确）
- 表现最差的类别（需要更多训练数据或数据增强）

### 5.2 问题类别诊断

自动按 F1 阈值分类：

| 等级 | 条件 | 含义 |
|------|------|------|
| 🔴 严重 | F1 < threshold/2 | 该类别几乎无法正确检测 |
| 🟡 关注 | F1 < threshold | 性能不理想，需要考虑改进 |
| 🟢 正常 | F1 ≥ threshold | 性能达标 |

### 5.3 类别级差异分析（多模型对比）

对比两个模型在每个类别上的 mAP50-95 差异：

```
| 类别     | model_a          | model_b          | 差异    | 胜出 |
|----------|------------------|------------------|---------|------|
| helmet   | 0.6500           | 0.5800           | +0.0700 | ←    |
| person   | 0.5500           | 0.4800           | +0.0700 | ←    |
```

### 5.4 评估趋势

追踪同一模型迭代训练过程中各项指标的变化趋势，自动计算改进/退步幅度。

## 六、补充建议

以下是可以进一步完善的方向，按优先级排列：

### 高优先级

1. **NMS/后处理参数扫描**：自动用不同的 conf/iou 组合跑评估，生成参数敏感度热力图，找最佳推理配置
2. **评估结果可视化**：在 `report.md` 中嵌入 matplotlib 生成的指标雷达图和柱状对比图
3. **CI 集成**：在 `.github/workflows/` 中添加 `odp-eval` 自动评估流水线，PR 合并前自动跑对比评估

### 中优先级

4. **TTA（测试时增强）评估**：在评估时自动启用测试时增强（翻转/多尺度），得到更稳健的指标
5. **错误样本导出**：将 FP/FN 最严重的样本图片打包导出，方便人工审核
6. **模型导出后评估**：支持评估 ONNX/TensorRT 导出模型的精度损失
7. **数据分布感知评估**：按图像尺寸/目标数量/目标大小分组评估，避免指标被某类数据主导

### 低优先级

8. **Web UI**：在 `apps/web` 中增加评估报告查看页面
9. **评估定时任务**：用 CronCreate 定期对新训练的模型自动评估
10. **多数据集批量评估**：一键在多个数据集上跑评估，生成综合健壮性报告
