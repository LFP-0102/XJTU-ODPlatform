## 一、MLflow 实验追踪

### 快速使用

```bash
pip install mlflow tensorboard
# 或: pip install -e "apps/platform[tracking]"

# 编辑 train.yaml → mlflow_enabled: true
odp-train

# 启动面板
mlflow ui --backend-store-uri sqlite:///mlflow.db
```

### 新增配置字段（`train.yaml` "实验追踪" 组）

`runtime_config/train_config.py` 新增 3 个 Pydantic Field，均写入 `FRAMEWORK_ONLY_FIELDS`（不传给 ultralytics）：

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `mlflow_enabled` | `bool` | `false` | 总开关 |
| `mlflow_tracking_uri` | `str\|null` | `null` | 存储后端（null=本地 `sqlite:///mlflow.db`） |
| `mlflow_experiment_name` | `str\|null` | `null` | 实验名（null=复用 `experiment_name`） |

### 架构设计

```
train_yolo()
├─ build_tracking_hooks_from_config()    ← 工厂：mlflow_enabled=False → 空钩子（零开销）
├─ fire_train_start()   → start_run + log_params（配置快照全量写入）
├─ _run_training()
│   ├─ model.add_callback("on_fit_epoch_end", ...)
│   └─ model.train() → 每轮自动触发 fire_fit_epoch_end() → log_metrics(step=epoch)
├─ fire_train_end()     → log_metrics(最终指标) + log_artifacts(best.pt) + end_run
└─ fire_train_error()   → end_run（异常清理）
```

**核心类**：

- `MlflowTracker` — MLflow 封装器。每个 API 调用独立 `try/except` + `logger.debug`，追踪失败不炸训练。未安装 mlflow 时 `_mlflow_available()=False`，全方法静默 no-op。
- `TrainTrackingHooks` — 训练生命周期钩子容器（对标 `model_infer/hooks.py::InferHooks`）。默认全 `None`，`fire_*` 全部 short-circuit，零开销。
- `build_tracking_hooks_from_config()` — 唯一入口，外部不接触 `MlflowTracker`。
- `_sanitize_key()` — 过滤 ultralytics 指标名中的括号（如 `metrics/mAP50(B)` → `metrics/mAP50B`），因为 MLflow 不接受 `()` 字符。

**service.py 改动**（6 处注入点）：

| 位置 | 改动 |
|---|---|
| import 区 | 导入 tracking 模块 6 个符号 |
| `_run_training()` 签名 | 新增 `tracking_hooks` 参数 |
| `_run_training()` 内 | 注册 ultralytics 回调；`mlflow_enabled=True` 时禁用 ultralytics 内置 MLflow |
| `train_yolo()` L116-129 | 构造 hooks → `fire_train_start` |
| `train_yolo()` KeyboardInterrupt | `fire_train_error` |
| `train_yolo()` Exception | `fire_train_error` |
| `train_yolo()` 训练成功后 | `fire_train_end` |

**`__init__.py` 改动**：从仅占位 docstring → 完整公共 API 文档 + 导入并重导出 7 个 tracking 符号。

### 5.1 新增：`model_train/tracking.py`（~305 行）

训练实验追踪核心模块，包含 4 个事件数据类 + 钩子容器 + MLflow 追踪器 + 工厂函数 + ultralytics 回调桥接。

```python
# ========== 4 个事件数据类 ==========
@dataclass(frozen=True)
class TrainStartEvent:
    """训练开始事件"""
    run_id: str; experiment_name: Optional[str]
    config_snapshot: Dict[str, Any]; data_yaml: str; model: str

@dataclass(frozen=True)
class FitEpochEndEvent:
    """每轮结束事件 —— ultralytics on_fit_epoch_end 回调触发"""
    epoch: int; metrics: Dict[str, float]

@dataclass(frozen=True)
class TrainEndEvent:
    """训练结束事件"""
    run_id: str; save_dir: Path
    best_weight: Optional[Path]; metrics: Any

@dataclass(frozen=True)
class TrainErrorEvent:
    """训练异常事件"""
    run_id: str; exception: Exception

# ========== 钩子容器 ==========
@dataclass
class TrainTrackingHooks:
    """对标 InferHooks: 4 个可选回调, 全部 None → 零开销"""
    on_train_start:    Optional[Callable[[TrainStartEvent],    None]] = None
    on_fit_epoch_end:  Optional[Callable[[FitEpochEndEvent],  None]] = None
    on_train_end:      Optional[Callable[[TrainEndEvent],      None]] = None
    on_train_error:    Optional[Callable[[TrainErrorEvent],    None]] = None

    def fire_train_start(self, evt):   # try/except 包裹, 回调异常不炸训练
    def fire_fit_epoch_end(self, evt): # 同上
    def fire_train_end(self, evt):     # 同上
    def fire_train_error(self, evt):   # 同上

# ========== MLflow 追踪器 ==========
class MlflowTracker:
    """每个 mlflow 调用独立 try/except, 追踪失败不影响训练.
       mlflow 未安装 → _mlflow_available()=False → 静默 no-op."""
    def __init__(self, tracking_uri=None, experiment_name=None):
        self._tracking_uri = tracking_uri or "sqlite:///mlflow.db"

    def start_run(self, run_id, tags=None):       # mlflow.start_run()
    def log_params(self, params):                 # mlflow.log_params() 分批(每批≤100)
    def log_metrics(self, metrics, step):         # ★ 自动 sanitize key 去括号
    def log_artifacts(self, paths):               # mlflow.log_artifact()
    def end_run(self):                            # mlflow.end_run()

    @staticmethod
    def _sanitize_key(key: str) -> str:
        """移除 mlflow 不允许的字符 (括号等)"""
        return key.replace("(", "").replace(")", "")

    @staticmethod
    def _mlflow_available() -> bool: ...          # 检测 mlflow 是否安装

# ========== 工厂 + 桥接 ==========
def build_tracking_hooks_from_config(config, run) -> TrainTrackingHooks:
    """唯一入口: mlflow_enabled=False → 空 hooks; mlflow 未安装 → 空 hooks
       否则创建 MlflowTracker + 4 个闭包回调"""
    # 闭包 _on_train_start:    tracker.start_run + log_params
    # 闭包 _on_fit_epoch_end:  tracker.log_metrics(step=epoch), 过滤不含 "/" 的 key
    # 闭包 _on_train_end:      tracker.log_metrics + log_artifacts + end_run
    # 闭包 _on_train_error:    tracker.end_run

def _make_ultralytics_callback(hooks) -> Dict[str, Callable]:
    """TrainTrackingHooks → ultralytics model.add_callback() 字典
       返回 {} 时 ultralytics 不注册任何回调, 零开销"""
```

### MLflow 指标名含括号导致 log_metrics 静默失败

- **现象**：MLflow 面板中 run 有 params 但 metrics 为空
- **根因**：ultralytics 指标键格式为 `metrics/mAP50(B)`、`metrics/precision(B)` 等，括号 `()` 不在 MLflow 允许字符集中，`mlflow.log_metrics()` 整个调用失败，被 `try/except` + `logger.debug` 吞掉
- **修复**：`MlflowTracker` 新增 `_sanitize_key()` 方法，`log_metrics()` 调用前自动去掉括号：`metrics/mAP50(B)` → `metrics/mAP50B`
- **位置**：`model_train/tracking.py:170-178`
